# 日线线段顶底识别程序

语言：中文 | [English](README.md)

这个项目把股票日线线段顶底识别方法整理成一个可运行的 Python 程序。入口是 `main.py`，核心判断在 `segment_analyzer/`。

程序会自行抓取日线数据，计算 MA5、MA34，输出所有连续线段的确认顶/底、候选顶/底、中枢、转折K和未确认原因。

## 环境

```bash
pip install -r requirements.txt
```

依赖：

- `pandas`
- `requests`
- `akshare`

日线数据默认使用缓存目录 `data/cache`。如果网络接口临时失败，程序会优先复用已有缓存。

## 运行方式

打开 GUI 窗口：

```bash
python3 main.py gui
```

窗口中输入股票代码即可分析。结果区会打印所有线段，并标注每条线段来源是 `中枢确认` 还是 `极值锚点`。分析完成后，可分别点击 `下载 TXT` 导出线段摘要，或点击 `下载 JSON` 导出完整详细信息。

分析指定股票：

```bash
python3 main.py stock --symbol 001259 --start 19900101 --end 20260529 --output out/001259.json
```

批量分析某天全部涨停股票：

```bash
python3 main.py limit-up --date 20260515 --lookback-days 520 --direction up --output out/zt_20260515.json
```

分析本地 CSV：

```bash
python3 main.py csv --file data/sample.csv --direction both --output out/sample.json
```

CSV 字段可以使用 `date/open/high/low/close`，也可以使用 AkShare 常见中文字段 `日期/开盘/最高/最低/收盘`。

常用参数：

- `--scan-mode lowest`：默认模式，从历史最低点作为确认底开始，向未来和过去补全线段。
- `--scan-mode history`：从历史左侧滑动发现线段。
- `--direction auto|up|down|both`：`history` 模式下用于控制初始方向。
- `--adjust ""|qfq|hfq`：复权方式，默认 `qfq`。
- `--max-segments`：最多识别多少条线段。
- `--cache-dir` / `--no-cache`：控制行情缓存。

## 代码结构

- `main.py`：命令行主程序。
- `segment_analyzer/data_fetcher.py`：抓取日线和涨停池。优先使用 AkShare；单股票日线有东方财富直连和腾讯日线接口兜底。
- `segment_analyzer/indicators.py`：标准化 OHLC，计算 MA5、MA34。
- `segment_analyzer/analyzer.py`：K0、中枢、起手三式、黑K、转折K、顶底四法则、端点延伸和全局后处理。
- `segment_analyzer/agent.py`：轻量调度和解释层，负责批量任务、输出 JSON、生成摘要。
- `segment_analyzer/models.py`：输出数据结构。

## 当前规则口径

### 扫描起点

默认 `lowest` 模式：

1. 找到全历史最低 Low，作为一个确认底。
2. 从最低点向未来按正时序寻找第一条向上线段。
3. 最低点之前的线段按左侧极值锚点补全：先找最低点之前的最高点并向下画线段，再找该最高点之前的最低点并向上画线段，循环直到剩余 K 线不足。
4. 所有线段保持连续：上一线段结束点就是下一线段开始点。

### K0

K0 是线段内用于寻找中枢线确认K的起始独立K。程序会保留所有 K0；每个候选中枢确认K判断时，取它前面最近的 K0 计算起手三式和 5K 重叠。

K0 的中枢区间重叠判断只和同一线段内前一个中枢有关，不和上一线段的中枢混用。

### 中枢确认K和中枢线

中枢确认K通过 MA5 反向突破逻辑筛选：

- 向上线段：实体最低价小于 MA5。
- 向下线段：实体最高价大于 MA5。

跳空规则：

- 中枢线确认K实体没有严格穿越 MA5 时，视为跳空。
- 普通跳空时，中枢线仍按实体下沿或上沿取值。
- 双跳空时，取实体上下沿中距离 MA5 最近的价格。

中枢线值由中枢确认K决定；中枢区间再由肉眼可见或非肉眼可见取轨规则决定。

### 起手三式

中枢成立需要满足起手三式之一：

- 反正两穿
- 三笔
- 5K 重叠

如果起手三式同时满足多个条件，选择更近的 K 作为成立K。

三笔统计可以早于 K0，但必须包含中枢确认K。

5K 重叠规则：

- 从中枢线确认K最近的 K0 往前找首K，截止到上一个转折K。
- 首K之后一根是破坏K，K0 也可以是破坏K。
- 取首K和破坏K的重叠区间；没有重叠时，用首K最高值到破坏K最低值形成区间。
- 从首K往后，K线价格区间 `[low, high]` 与重叠区间有交集即计数，边缘重叠也成立。
- 累计达到 5 根，且中枢确认K的最高/最低范围在重叠区间内，认为 5K 重叠成立。
- 最多允许一根跳空K计入 5K 重叠。
- 多个首K/破坏K组合满足时，选择离线段开始最近的组合。

### 黑K约束

黑K只在判断中枢是否成立时寻找一次。中枢成立后，顶/底候选只判断自身是否就是该唯一黑K。

黑K要求价格区间严格穿越 MA5，使用 K 线最高/最低值判断，不只看实体。

### 肉眼可见和非肉眼可见中枢

肉眼可见成立条件：

- 向上线段：确认K之后、下一根新中枢确认K出现之前，出现 MA5 小于确认K MA5 的 K 线。
- 向下线段：确认K之后、下一根新中枢确认K出现之前，出现 MA5 大于确认K MA5 的 K 线。

非肉眼可见判定：

- 从确认K开始，如果出现一根 K 的最大值大于确认K最大值，记录该 K。
- 后续若出现更大的最大值，更新记录。
- 出现三根最大值依次创新高且仍未满足肉眼可见条件，则按非肉眼可见中枢处理。

非肉眼可见取轨：

- 扫描范围从确认K往前，到前一转折K。
- 向上线段：找离转折K最近的相邻两根 K，价格区间有交集，且两根 K 的 high 都大于中枢线，另一条轨取两根 high 的较小值。
- 向下线段：找离转折K最近的相邻两根 K，价格区间有交集，且两根 K 的 low 都小于中枢线，另一条轨取两根 low 的较大值。

### 中枢延伸和重组

当前候选中枢成立之前，如果出现新的 K0 + 新确认K，会优先以新的组合重新分析。

中枢成立后，后续新的确认K才进入中枢延伸/重组判断。

重组需要满足：

- 存在 K 与当前中枢区间产生交互。
- 新确认K中枢线相对旧确认K中枢线按方向推进：
  - 向上线段：新确认K中枢线大于旧确认K中枢线。
  - 向下线段：新确认K中枢线小于旧确认K中枢线。
- 方向实体条件：
  - 向上线段：新确认K实体下沿大于旧确认K实体下沿。
  - 向下线段：新确认K实体上沿小于旧确认K实体上沿。

特殊情况：

- 旧中枢是非肉眼可见中枢，且新确认K满足延伸条件并能形成肉眼可见中枢时，新确认K的中枢替代旧中枢，视为重组。
- 旧中枢是肉眼可见中枢时，满足延伸条件的新确认K按延伸处理。
- 如果旧中枢已经能够确认顶/底，则后续延伸/重组不再生效。

### 转折K

转折K独立于顶/底判断。

- 候选顶的转折K条件：实体下沿小于 MA5。
- 候选底的转折K条件：实体上沿大于 MA5。
- 顶/底 K 本身不能是转折K；如果重合，该 K 视为转折K而不是顶/底。
- 一个转折K只分配给离它最近的唯一顶/底。
- 如果一个转折K已分配给某候选，早于该候选的候选会被忽略。

### 顶底四法则

法则1：独立2K

- 当前线段的顶底组合只要一侧满足独立2K即可。
- 判断独立2K时，候选K本身必须参与满足。
- 顶候选：候选K low 不低于中枢上轨，且三根K中至少两根 low 不低于中枢上轨。
- 底候选：候选K high 不高于中枢下轨，且三根K中至少两根 high 不高于中枢下轨。
- 线段中已经确认的顶/底需要根据当前线段新中枢重新判断独立2K。
- 左侧极值锚点的顶/底相关判断默认通过。

法则2：MA34/MA5

- 候选顶：
  - 顶K实体上沿大于 MA34。
  - 顶K对应 MA5 大于顶K对应 MA34。
- 候选底：
  - 底K实体下沿小于 MA34。
  - 底K对应 MA5 小于底K对应 MA34。

法则3：结构完美

只看当前线段要确认的一侧候选：

- 一般情况：
  - 顶K最低价不低于中枢上轨。
  - 底K最高价不高于中枢下轨。
- 如果候选顶/底不满足独立2K，且线段只有一个中枢：
  - 向上线段：顶K最高价大于中枢下轨。
  - 向下线段：底K最低价小于中枢上轨。
- 如果线段内超过一个中枢，按一般情况处理。

法则4：初始交叉

顶底组合之间必须存在 MA5/MA34 初始交叉点。MA5-MA34 从正到负、从负到正，或出现等于 0，都视为交叉。

### 端点延伸

一个线段确认后，优先寻找新的反向线段。如果新线段成立前出现满足条件的新确认顶/底，再进行当前端点延伸。

延伸规则：

- 向上线段只允许延伸到价格更高的确认顶，或转折K MA5 更高的确认顶。
- 向下线段只允许延伸到价格更低的确认底，或转折K MA5 更低的确认底。
- 因价格延伸时：
  - 向上线段：新顶实体上沿必须在当前顶实体下沿上方，重叠不算。
  - 向下线段：新底实体下沿必须在当前底实体上沿下方，重叠不算。
- 因转折K MA5 延伸时，不需要满足实体上下沿条件。
- 如果可延伸端点早于反向线段新端点，则先延伸，并从延伸后的端点继续判断。

### 全局完成后的共享端点替换

所有线段画完后，程序会再检查每条线段范围内的候选顶/底，用于修正共享端点。

判断范围：

- 取上一线段两个端点各自对应的转折K之间的 MA5 极值。
- 上一线段起点端点的转折K使用再上一线段的 `turn_k`。
- 上一线段终点端点的转折K使用上一线段自己的 `turn_k`。
- 如果缺少任一侧转折K，则不做该次替换判断。

替换规则：

- 向下线段：线段范围内候选顶满足四法则，且候选顶前后各2根K加候选K共 5 根中，任一根 MA5 严格大于上一线段转折K范围内 MA5 最大值，则用该候选顶替换当前顶。
- 向上线段：线段范围内候选底满足四法则，且候选底前后各2根K加候选K共 5 根中，任一根 MA5 严格小于上一线段转折K范围内 MA5 最小值，则用该候选底替换当前底。
- 多个候选满足时：
  - 向下线段选择候选顶K自身 MA5 最大的候选顶。
  - 向上线段选择候选底K自身 MA5 最小的候选底。
- 替换后会同步相邻线段共享端点，保持线段连续。

## 输出字段

JSON 输出中每条线段包含：

- `segment_no`、`segment_direction`：线段编号和方向。
- `confirmed_top` / `confirmed_bottom`：唯一确认顶底组合。
- `candidate_top` / `candidate_bottom`：所有 3K 顶底候选。
- `unconfirmed_candidates`：未被确认的候选及原因。
- `K0_index` / `K0_date`：当前线段使用的 K0。
- `pivot_confirm_k`、`pivot_line`、`pivot_lower`、`pivot_upper`、`pivot_type`：中枢信息。
- `start_pattern`：命中的起手三式。
- `has_black_k`、`diagnostics.black_k_refs`：黑K信息。
- `turn_k`：当前确认端点对应的转折K。
- `independent_2k_side`：独立2K满足侧。
- `initial_cross`：MA5/MA34 初始交叉点。
- `refresh_status`：端点延伸、锚点闭合或全局替换状态。
- `diagnostics`：中枢取轨、延伸边界、全局后处理替换等解释信息。
- `decision_status`：`confirmed`、`candidate`、`invalid`、`needs_more_k`。

## 注意事项

- 所有规则比较会先按 2 位小数取值。
- `name` 和 `trade_date` 只在外部传入或涨停池批量任务中有值；普通单股分析可以为空。
- 网络接口不稳定时，建议保留缓存或使用本地 CSV。
